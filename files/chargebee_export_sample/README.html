Chargebee

Invoices Export

Data in the exported files are comma separated and use the double quote (") character as the delimiter.

Note: Not all columns present in the exported files are explained here.

Invoices.csv

Invoice Number The invoice number. Acts as a identifier for invoice and typically generated sequentially.
Invoice Date Closing date of the invoice. Typically this is the date on which invoice is generated. If you have "wait & notify to send invoices enabled for usage based billing" this will not be the same as date on which invoice was actually sent to customer.
Start Date Start date of the invoice.
Status Current status of this invoice.
Next Retry Timestamp indicating when will the next attempt to collect payment for this invoice occur.
Recurring Boolean indicating whether this invoice belongs to a subscription
Tax Total Total tax amount for this invoice
Vat Number VAT/ Tax registration number of the customer. Learn more
PO Number Purchase Order Number for this invoice
Amount Due Total amount to be collected. This includes invoice's payments in progress.
Adjustments Total adjustments made against this invoice.
Credits Applied Total credits applied against this invoice.
Payments Total payments received for this invoice.
Write Off Amount Amount written off against this invoice.
Net Term Days Number of days within which the invoice has to be paid
Due Date Due date of the invoice
Dunning Status Current dunning status of the invoice.

Transactions.csv

Date Indicates when this transaction occurred.
Payment Method The Payment Method of this transaction
Amount Amount for this transaction.
Status The status of this transaction.
Authorization Reason Type of authorization transaction
Error Code Error code received from the payment gateway on failure.
Error Text Error message received from the payment gateway on failure.
Gateway Gateway through which this transaction was done. Applicable only for 'Card' Payment Method
Id At Gateway The id with which this transaction is referred in gateway.
Reference Number The reference number for this transaction. e.g check number in case of 'check' payments.
Voided At Timestamp indicating when the payment was voided or authorization expired at gateway.
Amount Unused Unused amount present for this transaction.
Amount Capturable Unused authorized/blocked amount for the transaction. Applicable only for Authorization transactions.
CreditNote Number Credit-note id.
Reference Authorization Id Authorization Transaction Id for which the payment transaction was created
Reference Transaction Id Payment Transaction Id for which the refund/reversal transaction was created
Gateway Account Id Identifier of the gateway

Customers.csv

Email Email of the customer. Configured email notifications will be sent to this email.
Billing Address Line1 Address line 1
Billing Address Line2 Address line 2
Billing Address City City
Billing Address State State or Province
Billing Address Country 2-letter ISO 3166 alpha-2 country code.
Billing Address Zip Zip or Postal code
Card Status Current status of the card.
Created At Timestamp indicating when this customer resource is created.
Billing Address State Code The ISO 3166-2 state/province code without the country prefix. Currently supported for USA, Canada and India. For instance, for Arizona (USA), set the state_code as AZ (not US-AZ). or, for Tamil Nadu (India), set the state_code as TN (not IN-TN). or, for British Columbia (Canada), set the state_code as BC (not CA-BC).
Note: If the 'state_code' is specified, the 'state' attribute should not be provided as Chargebee will set the value automatically (for US, Canada, India).
Customer Portal Status Status of the customer portal.
Net Term Days The number of days within which the customer has to make payment for the invoice
Taxability Specifies if the customer is liable for tax
Locale Determines which region-specific language Chargebee uses to communicate with the customer. In the absence of the locale attribute, Chargebee will use your site's default language for customer communication.
Billing Address First Name First name
Billing Address Last Name Last name
Billing Address Email Email
Billing Address Company Company name
Billing Address Phone Phone number
Billing Address Line3 Address line 3
Billing Address Validation Status The address verification status.
Is Location Valid Customer location is validated based on IP address and Card issuing country. If the location is valid, it returns True. If not, it returns False. Applicable only for EU, New Zealand and Australia.
VAT Number Status This is only applicable if you have configured EU and Australian taxes and the VAT number validation is enabled.
VAT Number Validated Time VAT number recently got validated time

Subscriptions.csv

Plan Id A unique ID for your system to identify the plan.
Plan Quantity Represents the plan quantity for this subscription.
Status Current state of the subscription
Start Date Applicable only for 'future' subscriptions. The scheduled start time of the 'future' subscription.
Trial Start Start of the trial period for the subscription. Presence of this value for 'future' subscription implies the subscription will go into 'trial' state when it starts.
Trial End End of the trial period for the subscription. Presence of this value for 'future' subscription implies the subscription will go into 'trial' state when it starts.
Curr Term Start Start of the current billing term.
Curr Term End End of the current billing term. Subscription is renewed immediately after this
Next Billing Date on which the next billing happens.
Remaining Billing Cycles
  • When the subscription is on a contract term: this value is the number of billing cycles remaining in the contract term after the current billing cycle.
  • When the subscription is not on a contract term: this value is the number of billing cycles remaining after the current cycle, at the end of which, the subscription cancels.
Created At Subscription created time
Started At Time at which the subscription got started. Will be null for 'future' subscriptions as it is yet to be started.
Activated At Time at which the subscription moved from in_trial state to active state
Cancelled At Time at which subscription was cancelled or is set to be cancelled.
Cancel Reason Possible reason the subscription might be cancelled.
Next Billing Amount Total invoice amount to be paid upon next renewal.
PO Number Purchase Order Number for this subscription
MRR Monthly Recurring Revenue
Subscription Plan Unit Price Amount that will override the Plan's default price.
Subscription Setup Fee Amount that will override the default setup fee.
Subscription Pause Date Date on which the subscription will be paused.
Subscriptions Resume Date Date on which the subscription will be resumed.

SubscriptionAddons.csv

Addon Id A unique ID for your system to identify the addon.
Addon Quantity Addon quantity. Applicable only for the quantity based addons. Should be passed with the same index as that of associated addon id.
Addon Unit Price Amount that will override the Addon's default price. The Plan's billing frequency will not be considered for overriding. E.g. If the Plan's billing frequency is every 3 months, and if the price override amount is $10, $10 will be used, and not $30 (i.e $10*3).

SubscriptionCoupons.csv

Coupon Id Used to uniquely identify the coupon in your website/application and to integrate with Chargebee.
Coupon Code Id Unique coupon code that can be redeemed only once.
Redemptions The number of times this coupon has been redeemed.
Apply Till The date till the coupon is to be applied. Applicable for "limited months" coupons

Pricing Model Charges a single price for the plan/addon

Quantity Used The number of units purchased in a range.
Unit Amount The price of the tier if the charge model is a stairtstep pricing , or the price of each unit in the tier if the charge model is tiered/volume pricing.

Plans.csv

Plan Id A unique ID for your system to identify the plan.
Name The display name used in web interface for identifying the plan.
Invoice Name Display name used in invoice. If it is not configured then name is used in invoice.
Price The price of the plan
Period Defines billing frequency. Example: to bill customer every 3 months, provide "3" here.
Period Unit Defines billing frequency in association with billing period.
Trial Period The free time window for your customer to try your product
Trial Period Unit Time unit for the trial period.
Setup Cost One-time setup fee charged as part of the first invoice.
Free Quantity Free quantity the subscriptions of this plan will have. Only the quantity more than this will be charged for the subscription.
Billing Cycles The number of billing cycles the subscription is active. The subscription is moved to non renewing state and then to cancelled state automatically
Redirect Url The url to redirect on successful checkout. Eg: https://yoursite.com/success.html?plan=basic
Status The plan state
Archived At Time at which the plan was moved to archived status.
Downgrade Penality Penalty charge is calculated by % entered with the actual value of the plan for the duration used.
Charge Model Defines how the subscription recurring charge for this plan should be calculated.
Description Description about the plan to show in the hosted pages & customer portal.
SKU The field is used as Product name/code in your third party accounting application. Chargebee will use it as an alternate name in your accounting application.
Accounting Code This field is to capture the Account code setup in your Accounting system for integration purposes only.
Accounting Category1 The name of the category of your product in Xero. If you've integrated with QuickBooks, this will be the "Class". Use the format "<Category>:<Name>". E.g. "Region: North".
Accounting Category2 The name of the category of your product in Xero. Use the format<Category>:<Name>". E.g. "Region: North"
